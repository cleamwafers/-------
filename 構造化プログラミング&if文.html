構造化プログラミング

下の３つの構造を組み合わせてプログラムを書くこと。プログラムがわかりやすくなる。→作りやすくなる。間違いにくくなる。メンテナンスしやすくなる。

  1. 順次進行

    2. 上から順番に処理を行う。

  2. 分岐(Javaではif文とswitch文)

    3. ある条件が満たされるならこの処理をし、満たされないなら別の処理をする。=処理を分ける。

  3. 繰り返し(Javaではwhole文とfor文)

    4. ある条件が満たされている間は同じ処理を繰り返す。

if文の書き方

if(条件式){

処理１

}else{

処理２

}

ポイント
  • どちらかの処理しか実行されない
  • else以降は省略可
  • 処理が一つのときは{}を省略可

    • else-if文の書き方

    if(条件式1){

    処理１

    }elseif(条件式2){

    処理２

    }else{

    処理3

    }

    ポイント
  • 条件式は上から順に判定

    • switch文

    switch(式){

    case 値1:

    処理１

    break;

    case 値２:

    処理2

    break;

    default:

    処理3

    }

    注意点
  • 式はbyte、short、int、charのみ

    • 整数の値が文字

  • 式の値によって処理の開始位置が決まる
  • breakは{}を抜ける
    • switch文とif文どう使い分けるのか？

    switch文で書けることはif文でも書ける

    if文で書けることがswitch文で書けないパターンがある

    →if文の方が複雑な分岐処理が書けるので普通はif文を使う

    但し、変数の値によって処理を一度に多分岐したいときはswitch文を使うことでプログラムが分かりやすくなる。
